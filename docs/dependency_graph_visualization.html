PTO Runtime 依赖图动态构建

理解任务图如何在运行时通过 TensorMap 和哈希查找动态构建

核心算法：依赖是如何动态构建的

📌 依赖构建算法总览 增量式 DAG 构建

PTO Runtime 采用增量式方法构建任务依赖图：每次提交任务时，通过查找 TensorMap 发现依赖，然后建立双向链接。

  1. 任务提交 pto2_submit_task() 被调用 O(1)
  2. 遍历 INPUT 参数：对每个输入张量执行查找
  3. TensorMap 查找：用哈希表找到"谁生产了这个张量" O(n)
  4. 去重检测：避免同一生产者被重复添加 O(n²)
  5. 建立依赖链接：consumer.fanin ← producer, producer.fanout → consumer O(1)
  6. 注册 OUTPUT：将输出张量插入 TensorMap，供后续任务查找 O(1)

为什么是"动态"构建？

传统框架（如 TensorFlow 1.x）需要先定义完整计算图，然后执行。

PTO Runtime 是动态的：

  • 任务图在运行时逐个任务构建
  • 提交 Task C 时，通过 TensorMap 查找发现它依赖 Task B
  • 不需要预先知道完整的图结构
  • 支持条件分支、循环等动态控制流

TensorMap 的作用

TensorMap 是一个地址 → 生产者的映射表：

  • Key：张量的内存地址（buffer.addr）
  • Value：生产该张量的任务 ID
  • 当 Task B 提交时，用 INPUT 地址查找，找到 Task A
  • 然后建立 A → B 的依赖关系

性能热点（红色边框步骤）

  • Step 2：哈希链遍历，链过长时 O(n)
  • Step 3：线性搜索去重，O(n²) 复杂度
  • Step 4：自旋锁竞争，广播场景下严重

TensorMap.lookup() 算法 链地址法哈希查找

// 查找张量的生产者任务 void lookup(const Tensor& query, PTO2LookupResult& result) { result.count = 0; // Step 1: 计算哈希值 → 定位桶 uint32_t bucket_idx = hash(query.data().buffer.addr); // Step 2: 获取链表头 PTO2TensorMapEntry* cur = buckets[bucket_idx]; // Step 3: 遍历链表（链地址法） while (cur != nullptr) { // 检查条目是否过期（惰性清理） if (!entry_valid(*cur)) { remove_stale_entries(cur); return; } // Step 4: 检查地址匹配 if (query.data().buffer.addr == cur->addr) { // Step 5: 检查内存重叠 if (query.is_overlap(cur->tensor.data())) { result.push(cur); // 找到依赖! } } cur = cur->next_in_bucket; // 链表下一节点 } }

依赖链构建算法 双向链表 + 去重

// submit_task 中的依赖构建 // 1. 临时数组收集 fanin int32_t fanin_temp[PTO2_MAX_INPUTS]; int32_t fanin_count = 0; // 2. 遍历每个 INPUT 参数 for (int i = 0; i < num_params; i++) { if (params[i].type == INPUT) { // 查找 TensorMap PTO2LookupResult result; tensor_map.lookup(params[i].tensor, result); for (int r = 0; r < result.count; r++) { int32_t producer_id = result.entries[r]->producer_task_id; // 🔴 O(n²) 去重检测 - 性能瓶颈! bool duplicate = false; for (int j = 0; j < fanin_count; j++) { if (fanin_temp[j] == producer_id) { duplicate = true; break; } } if (!duplicate) { fanin_temp[fanin_count++] = producer_id; // 添加到 producer 的 fanout 列表 pto2_add_consumer(producer_id, task_id); } } } } // 3. 构建 fanin 链表 for (int i = 0; i < fanin_count; i++) { task->fanin_head = dep_list_prepend(task->fanin_head, fanin_temp[i]); }

TensorMap.insert() 算法 头插法 O(1)

void insert(const Tensor& tensor, int32_t producer_id) { // Step 1: 分配条目 PTO2TensorMapEntry* entry = new_entry(); entry->producer_task_id = producer_id; entry->addr = tensor.data().buffer.addr; // Step 2: 头插法插入哈希桶 (O(1)) uint32_t bucket_idx = hash(entry->addr); entry->next_in_bucket = buckets[bucket_idx]; buckets[bucket_idx] = entry; // Step 3: 链接到任务的条目列表（用于批量清理） int32_t slot = producer_id & (TASK_WINDOW_SIZE - 1); entry->next_in_task = task_entry_head[slot]; task_entry_head[slot] = entry; }

内存重叠检测

TensorMap 不仅检测精确地址匹配，还检测内存区域重叠，支持 view、reshape 等操作。

PARTIAL_OVERLAP 部分重叠

张量 B

B 部分覆盖 A → B 依赖 A

A
